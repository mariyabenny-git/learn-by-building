Student Portfolio
A personal profile page that loads all content from a JSON file. Change the JSON, the page updates.
Folder Structure
📁 student-portfolio/
  ├── index.html
  ├── style.css
  ├── script.js
  └── profile.json
JSON Schema
{ "name": "Your Name", "bio": "Short intro about yourself", "avatar": "https://link-to-photo.jpg", "skills": [ { "name": "HTML", "level": "Beginner" } ], "projects": [ { "title": "My First App", "description": "What it does", "image": "screenshot.png", "link": "https://github.com/..." } ], "socialLinks": [ { "platform": "GitHub", "url": "https://..." } ] }
Task Checklist
  1. 1 Set up the 4 files — index.html, style.css, script.js, profile.json
  2. 2 Build the HTML skeleton — use semantic tags: header, nav, main, section, footer
  3. 3 Fill profile.json with your own data — name, bio, skills, projects, links
  4. 4 In script.js, use fetch() to load profile.json and render all sections into the DOM
  5. 5 Style the page — use flexbox for nav and skills, CSS grid for project cards
  6. 6 Add media queries so it works on mobile (stack cards, collapse nav)
  7. 7 Add interactions — smooth scroll, hover effects, form validation on contact section
  8. 8 Push to GitHub and deploy on GitHub Pages
Bonus Challenges
  • Dark / light theme toggle using CSS variables
  • Scroll-based active nav link highlighting
  • A downloadable resume button that links to a PDF
Recipe Book
A browsable recipe collection. Filter by category, search by name, click to view the full recipe.
Folder Structure
📁 recipe-book/
  ├── index.html
  ├── style.css
  ├── script.js
  └── recipes.json
JSON Schema
{ "recipes": [ { "id": 1, "name": "Paneer Butter Masala", "category": "Vegetarian", "image": "https://link-to-image.jpg", "prepTime": "30 mins", "ingredients": [ "200g paneer", "2 tomatoes" ], "steps": [ "Heat oil in a pan...", "Add chopped tomatoes..." ] } ] }
Task Checklist
  1. 1 Set up the 4 files — index.html, style.css, script.js, recipes.json
  2. 2 Add 5-6 recipes to your JSON with different categories (Veg, Non-Veg, Dessert)
  3. 3 Build the HTML — a search bar, category filter buttons, and a container for recipe cards
  4. 4 Fetch recipes.json and render each recipe as a card (image, name, category, prep time)
  5. 5 Add category filter — clicking a button shows only recipes of that category
  6. 6 Add search — filter cards in real-time as the user types in the search bar
  7. 7 Add a detail view — clicking a card shows full ingredients and steps (modal or new section)
  8. 8 Style everything — card layout with grid, hover effects, responsive for mobile
Bonus Challenges
  • Favorites list using localStorage — heart icon on each card
  • Print-friendly view for a single recipe (use @media print)
  • Clickable ingredient checklist in the detail view
Quiz App
A multiple-choice quiz that loads questions from JSON, tracks score, and shows results at the end.
Folder Structure
📁 quiz-app/
  ├── index.html
  ├── style.css
  ├── script.js
  └── questions.json
JSON Schema
{ "questions": [ { "id": 1, "question": "What does HTML stand for?", "options": [ "Hyper Text Markup Language", "Hot Mail", "How To Make Lasagna", "Hyperlinks Text Mark Language" ], "correctAnswer": 0, // index of correct option "explanation": "HTML = Hyper Text Markup Language" } ] }
Task Checklist
  1. 1 Set up the 4 files — index.html, style.css, script.js, questions.json
  2. 2 Add 8-10 questions to your JSON — use topics you're studying (CS basics, web dev, etc.)
  3. 3 Build the HTML — a start screen, question display area, option buttons, and a results section
  4. 4 Fetch questions.json and show one question at a time with its options
  5. 5 Handle answer selection — highlight correct/wrong, show explanation, then move to next
  6. 6 Track the score in a JS variable, show progress (e.g., "Question 3 of 10")
  7. 7 Show a results screen at the end — total score, percentage, and a restart button
  8. 8 Style it — clean card layout, color feedback for correct/wrong, transitions between questions
Bonus Challenges
  • Countdown timer per question (e.g., 15 seconds)
  • Shuffle question and option order on each attempt
  • Score history saved in localStorage with date and time
General Tips
  • Use browser DevTools (F12) — inspect elements, check console for errors, test responsiveness
  • Commit to git after every checklist item — build the habit early
  • Don't copy-paste code blindly — type it out, understand what each line does
  • Deploy on GitHub Pages — a live link is worth more than code on your laptop
  • When stuck, read MDN Web Docs — it's the best reference for HTML, CSS, and JS
